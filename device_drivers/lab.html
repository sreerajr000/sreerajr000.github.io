List all running daemon processes in a Linux System

$ ps -eo 'tty,pid,comm' | grep ^?

SuperUser Commands and User Commands

User Commands

1. wget
A utility for non-interactive download of files from the Web. It supports HTTP, HTTPS, and FTP protocols, as well as retrieval through HTTP proxies.
$ wget https://sreerajr000.github.io/abcd.zip

2. ping
Ping a remote host by sending only 5 packets.
$ ping -c 5 gmail.com

3. less
Less does not have to read the entire input file before starting, so with large input files it starts up faster than text editors like vi
$ less sys.log

4. locate
Using locate command you can quickly search for the location of a specific file (or group of files). Locate command uses the database created by updatedb.
$ locate crontab

5. uname
Uname command displays important information about the system such as â€” Kernel name, Host name, Kernel release number, Processor type, etc.
$ uname -a

6. passwd
Change your password from command line using passwd. This will prompt for the old password followed by the new password.
$ passwd USER

7. chown
chown command is used to change the owner and group of a file.
$ chown oracle:dba dbora.sh

8. df
Displays the file system disk space usage. By default df -k displays output in bytes.
$ df -k

9. free
This command is used to display the free, used, swap memory available in the system.
$ free

10. service
Service command is used to run the system V init scripts. i.e Instead of calling the scripts located in the /etc/init.d/ directory with their full path, you can use the service command. Check the status of a service:
$ service ssh status

11. sort
Sort a file in ascending order
$ sort abc.txt

12. touch
The touch command - a.k.a. the make file command - allows users to make files using the Linux CLI. Just as the mkdir command makes directories, the touch command makes files. Just as you would make a .doc or a .txt using a PC desktop, the touch command makes empty files. An example of the touch command:
$ touch abc.txt

13. clear
The clear command does exactly what it says. When your Linux CLI gets all mucked up with various readouts and information, the clear command clears the screen and wipes the board clean. Using the clear command will take the user back to the start prompt of whatever directory you are currently operating in. To use the clear command simply type clear.
$ clear

14. chmod
Chmod changes the access mode (permissions) of one or more files. Only the owner of a file or a privileged user may change the access mode. See examples of changing the permissions of files using chmod.
$ chmod +x run.sh

15. netstat
Netstat provides information and statistics about protocols in use and current TCP/IP network connections. It is a helpful forensic tool in figuring out which processes and programs are active on a computer and are involved in network communications.
$ netstat


SuperUser Commands

1. apt
apt provides a high-level commandline interface for the package management system. It is intended as an end user interface and enables some options better suited for interactive usage by default compared to more specialized APT tools like apt-get.
# apt-get install python

2. fsck
fsck is used to check and optionally repair one or more Linux filesystems. filesys can be a device name (e.g. /dev/hdc1, /dev/sdb2), a mount point (e.g. /, /usr, /home), or an ext2 label or UUID specifier
# fsck -nf /dev/sda1 -y

3. badblocks
badblocks is used to search for bad blocks on a device (usually a disk partition).
# badblocks /dev/sda1

4. parted
parted is a program to manipulate disk partitions. It supports multiple partition table formats, including MS-DOS and GPT. It is useful for creating space for new operating systems, reorganising disk usage, and copying data to new hard disks.
# parted /dev/sda

5. umount
umount - unmount file systems
# umount /mnt

6. kbdrate
kbdrate is used to change the keyboard repeat rate and delay time. The delay is the amount of time that a key must be depressed before it will start to repeat.
# kbdrate

7. hwclock
hwclock is a tool for accessing the Hardware Clock. It can: display the Hardware Clock time;
# hwclock

8. systemctl
systemctl may be used to introspect and control the state of the "systemd" system and service manager.
# systemctl restart network-manager.service

9. dpkg
dpkg is a tool to install, build, remove and manage Debian packages.
# dpkg -i abc.deb

10. slabtop
Display kernel slab cache information in real time Check the status of a service:
# slabtop

11. adduser
adduser, addgroup - add a user or group to the system
$ adduser [OPTIONS]

12. runuser
runuser allows to run commands with a substitute user and group ID.
$ runuser [OPTIONS]

13. clear

Using the clear command will take the user back to the start prompt of whatever directory you are currently operating in. To use the clear command simply type clear.
